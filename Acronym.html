Never learn an Acronym again.

admin 9 January 2017 18 js, php, html permalink

As a Computer Science student and an IT professional, I deal with acronyms, terminal commands and other IT lingos on a regular, particularly with Networking and System Administration duties. They are the field’s own jargon to describe protocols, processes, standard etc... I thought it might be a good idea to make some research and read on the topic to get a bit of familiarity As I was getting ready for a Linux Systems Administration class, I ended up writing a small program that learns my acronyms and jargon so that I don’t have to.

I am not talking about machine learning, web scraping or any complex algorithm of sort. It is just an ordinary directory with text files each containing the definition of an acronym.

The idea came to me after I purchased the “Unix and Linux System Administration Hanbook 5th edition” ebook by Nemeth, it has quite positive reviews online, and seems to be well appreciated in the Sysadmin community. It was a required class material; I didn’t really have a choice Anyways. When I started reading the book, I had a bad first impression, or maybe, I should just say that couldn’t keep up with the number of acronyms I had to deal with, they were everywhere, every paragraph of every chapter. From NFS, SUS, GRUB to RPC, LLVM and ACL. I started thinking to myself there is no way I will ever be able to remember all of these.

I tried writing them down but at times it felt quite a cumbersome, like when I’m stuck and needed a definition from previous chapters, it would take me a few minutes either to swipe back to a specific page or to find where it was save in my note app. Sometimes I wouldn’t even find it at all, either because I couldn’t remember where it was saved or maybe I didn’t save it at first place, who knows! My frustration started to grow; I needed a way around this issue. That is when I decided to write a program that would learn my acronyms for me.



Lazyness is a virtue

I was thinking of something similar to the Unix manual command “man” for instance. I use it on a daily basis, it is quite handy, nothing special to those of you who are familiar with the command line environment, whenever you need information regarding any Unix command, you’d simply type: 

						    $ man "the command"

the terminal would display useful information related to that command, no need to memorize anything at all. I even use this great tool when I am taking an exam, it is fast and reliable.

How awesome would it be if I could simply type 

$ man

and get the definition and usage right from the terminal?

That is quite unfortunate, this would not work. But I needed sometime as simple; I wanted a practical keyword like “what is”. Say I type the following on my terminal: 

“what is LLVM”

and the program would print out:

 “ > LLVM: Low Level Virtual Machine.”

Sounds like a great idea, but I couldn’t use “whatis” because, as you probably know, it is already a Unix system command. The choice of the keywords to trigger the application was crucial as I wanted it to be intuitive and fun to use.

Then I got inspired by the instant messaging lingos trends. I remember how I always find myself using abbreviated terminologies like, wyd, brb, rofl, wht or whts up when texting. I then decided to use the keyword “whts” which I assume stands for “what is” and more importantly is not a Terminal command.

Going back to the previous LLVM example, we would type:

$ whts LLVM”

and the program would print out:

“> LLVM: Low Level Virtual Machine.”

Cool right??

Good! I have also opted to write the source code in C, I know many of you would argue why not Python or a simple shell script? But this is not the point here, the tool doesn’t really matter! as long as the program works, I am be one step closer to pass a class.

Further, I figured that I need to tell the program when and how to add, remove, search and print an acronym or to display program documentation. I decided to rely on command-line switches to determine the application behavior, I could use an option switch that would tell the program exactly what I want it to do with the acronym and how to get it done.

For instance, If I want to edit our LLVM definition, I could type:

$ whts -e LLVM

and it would open the acronym definition in a way to let me edit and save the changes. 

 > LLVM is an acronym that stands for Low Level Virtual Machine.
